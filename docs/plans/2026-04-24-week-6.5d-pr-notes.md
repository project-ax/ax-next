## Invariants (I1ā€“I9)

Carry-forwards from 6.5a's deviations, enumerated as numbered invariants in the plan so the reviewer can audit against them.

- **I1 ā€” runner holds no LLM credentials.** The sandbox never sees a real `ANTHROPIC_API_KEY`. The env var passed to claude-sdk IS the per-session bearer token (same token that authenticates IPC). The proxy validates it via `session:resolve-token`. Enforced by `packages/agent-claude-sdk-runner/src/main.ts` (sets `ANTHROPIC_API_KEY: env.authToken`) + `packages/llm-proxy-anthropic-format/src/listener.ts` (bearer check before body read).
- **I2 ā€” schema source of truth stays in `@ax/ipc-protocol`.** No new IPC actions in 6.5d. `tool.execute-host` (schema-only in 6.5a) earns runtime wiring here; `packages/agent-claude-sdk-runner/src/host-mcp-server.ts` forwards to it.
- **I3 ā€” proxy HTTP surface is a new attack surface.** Bound to `127.0.0.1:<ephemeral>`; `Authorization: Bearer` check runs BEFORE any body read; body-size cap = 4 MiB (matches `@ax/ipc-server`); auth token never echoed. Enforced by `packages/llm-proxy-anthropic-format/src/listener.ts` + covered by 22 listener tests.
- **I4 ā€” no half-wired plugins.** Both new packages are reachable from the acceptance test. The runner binary is resolved in `@ax/cli` via `createRequire` (same mechanism as `@ax/agent-native-runner`); the proxy plugin is registered conditionally when `runner === 'claude-sdk'`.
- **I5 ā€” honest tool name passthrough.** The claude-sdk tool name (`Bash`, `Read`, `mcp__ax-host-tools__<name>`, ā€¦) flows verbatim into `tool:pre-call` / `tool:post-call` subscribers. No remap. Enforced by `packages/agent-claude-sdk-runner/src/tool-names.ts` (classifier) + asserted by the acceptance test (`preCallEvents.map((e) => e.call.name)` equals `['Bash', 'mcp__ax-host-tools__test-host-echo']`).
- **I6 ā€” no new dependency on sandbox-side impl packages.** claude-sdk ships its own `Bash`/`Read`/`Write`/`Edit`. `@ax/tool-bash-impl` and `@ax/tool-file-io-impl` stay wired to `@ax/agent-native-runner` only. Enforced by `packages/agent-claude-sdk-runner/package.json` (no such deps).
- **I7 ā€” turn boundary = `SDKResultMessage`.** `event.turn-end` and `event.chat-end` fire on the SDK's terminal `result` message, never mid-stream. Enforced by `packages/agent-claude-sdk-runner/src/main.ts` (`msg.type === 'result'` branch).
- **I8 ā€” streaming SSE is synthesized, not pass-through.** IPC `llm.call` is non-streaming. The proxy issues one IPC call per `/v1/messages` request and emits faux SSE frames. Enforced by `packages/llm-proxy-anthropic-format/src/sse-frames.ts` + covered by dedicated tests.
- **I9 ā€” proxy listener lifecycle mirrors IPC listener.** Per-session bind, started alongside `ipc:start` and torn down alongside `ipc:stop`. Proxy-start failure rolls back `ipc:stop` + `session:terminate` + tempdir cleanup (same shape as `ipc:start` rollback). Enforced by `packages/sandbox-subprocess/src/open-session.ts` + covered by rollback test.

## Scope decisions

The 10 locked-in calls from the plan, each with a one-line status.

1. **Streaming vs non-streaming proxy.** Both supported. Non-streaming = direct JSON response; streaming = synthesized SSE frames from one bulk `llm:call`. Token-level re-streaming from the provider is not implemented (that was 6.5b, skipped).
2. **Tool name convention.** claude-sdk names pass verbatim to host subscribers (`Bash`, `mcp__ax-host-tools__<name>`). The runner strips the `mcp__ax-host-tools__` prefix only when dispatching to `tool.execute-host` (host expects bare name).
3. **MCP server grouping.** One `createSdkMcpServer({ name: 'ax-host-tools' })` per runner; all `executesIn: 'host'` tools register into it.
4. **Disabled built-ins.** `disallowedTools: ['WebFetch', 'WebSearch', 'Skill', 'Task']`. Deny-list, not allow-list (fragile when SDK adds built-ins).
5. **`settingSources: []`.** AX owns the context; no CLAUDE.md / project-settings bleed-through. Pinned by test in `3392638`.
6. **System prompt.** `{ type: 'preset', preset: 'claude_code' }`. The runner's job is to be claude-sdk; reinventing the prompt would defeat the point.
7. **`pathToClaudeCodeExecutable`.** Unset. The SDK bundles its own claude-code CLI; pnpm hoisting makes it reachable from the sandbox's `node_modules`.
8. **Proxy auth scheme.** Reuses the existing session bearer token. No new auth boundary. The proxy calls `session:resolve-token`, same path `@ax/ipc-server` uses.
9. **Proxy transport.** Plain HTTP on `127.0.0.1:<ephemeral>` per session, URL injected via `AX_LLM_PROXY_URL` ā†’ claude-sdk's `ANTHROPIC_BASE_URL`. k8s / Docker sandboxes will need to re-route (Week 7ā€“9, flagged below).
10. **Built-in vs MCP `tool:pre-call` parity.** Both paths fire `tool:pre-call` with `{id, name, input}` and respect the verdict. `canUseTool` synthesizes a runner-local UUID for the pre-call (no `tool_use_id` available at that stage); `PostToolUse` emits the real `tool_use_id`. We do not expose a preā†”post correlation contract in 6.5d (post is observation-only per D4).

## Boundary review

### `llm-proxy:start` / `llm-proxy:stop` (`@ax/llm-proxy-anthropic-format`)

- **Alternate impl:** `@ax/llm-proxy-mock` (tests) or `@ax/llm-proxy-openai-format` (future provider). The start/stop surface is intentionally format-agnostic.
- **Payload fields:** `sessionId` (canonical), `url` (generic HTTP URL), `port` (numeric). No Anthropic-specific names leak.
- **Subscriber risk:** N/A ā€” service hooks, no subscribers.
- **Wire surface:** the proxy's `POST /v1/messages` IS a wire surface, but it's an external API (Anthropic Messages spec), not an AX IPC action. `@ax/ipc-protocol` schemas unaffected. The Anthropic request/response Zod schemas live in `packages/llm-proxy-anthropic-format/src/anthropic-schemas.ts`.

### `sandbox:open-session` extension (new `calls:` entries)

`@ax/sandbox-subprocess` now declares `calls: ['llm-proxy:start', 'llm-proxy:stop']` on its manifest. The session-lifetime contract is unchanged; the extension is additive (best-effort `stop` on close, rollback on failed `start`). No boundary-review re-review needed per the CLAUDE.md "internal-only changes" carve-out ā€” the hook surface itself is owned by `@ax/llm-proxy-anthropic-format`.

## Open issues / known gaps

Called out explicitly so the reviewer doesn't have to dig.

1. **Branch target is `main`, not 6.5c.** 6.5b (token-level streaming) and 6.5c skipped for MVP per `project_mvp_direction.md`. Plan-file noted this; calling it out here too.
2. **`llm:pre-call` / `llm:post-call` subscribers are NOT wired at runtime.** The plan defers these to "a future slice when the first subscriber (canary) earns its keep." The proxy calls `llm:call` directly; no transformation chain.
3. **Zod peer-dep mismatch.** Workspace uses `zod@^3.23.8`; `@anthropic-ai/claude-agent-sdk@0.2.119` declares `zod@^4.0.0` as a peer. At runtime the SDK works against zod@3's surface (we exercise it in the acceptance test); documented in `packages/agent-claude-sdk-runner/SECURITY.md`. Upgrading the whole workspace to zod@4 is a separate slice.
4. **k8s / Docker sandbox plugins will need to re-route proxy HTTP.** The proxy binds to `127.0.0.1` on the host; subprocess sandbox inherits the host's network namespace, so this works today. k8s pod-per-session (Week 7ā€“9) will have to surface the proxy inside the pod's namespace ā€” either by sidecar or by a cluster-HTTP URL in place of loopback. Not blocking MVP.
5. **Task 13 `test-host-echo` stub relies on the dispatcher passing the full `{id, name, input}` envelope.** Candidate for a documented boundary contract when the first real host-mediated tool plugin lands (likely 6.5e `mcp-client`). Not a bug today ā€” the contract is enforced by the acceptance test ā€” but worth codifying when a second consumer arrives.
