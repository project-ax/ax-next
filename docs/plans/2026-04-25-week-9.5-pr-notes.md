# Week 9.5 — multi-tenant model PR notes

Closes the multi-tenant primitives (auth + agents + admin API) for Week 9.5 per `docs/plans/2026-04-24-week-9.5-multi-tenant-handoff.md`.

## Summary

- New plugins: `@ax/http-server` (pulled forward from Week 10-12), `@ax/auth` (Google OIDC + dev-bootstrap + cookie sessions), `@ax/agents` (entity + ACL), `@ax/teams` (membership + ACL).
- Admin API surface: `/admin/agents`, `/admin/mcp-servers`, `/admin/teams`, `/admin/me`, `/admin/sign-out`. Auth API: `/auth/sign-in/:provider`, `/auth/callback/:provider`, `/auth/sign-out`, `/auth/dev-bootstrap`.
- Chat path now resolves an agent (with ACL gate) before opening a sandbox; the runner reads its frozen agent config via a new `session.get-config` IPC action; `tool:list` is filtered to the agent's `allowedTools` + `mcpConfigIds`.
- One-shot CLI: `ax-next admin bootstrap` mints the first admin against a freshly-booted preset.
- Two enforcement rails: `no-restricted-imports` blocks cross-plugin imports; a custom `local/no-bare-tenant-tables` rule blocks `selectFrom('agents_v1_*' | 'auth_v1_*' | 'teams_v1_*')` outside `store.ts` / `scope.ts` / `__tests__/`.

## Security review (consolidated)

### Sandbox

Net-new capabilities are minimal and bounded.

- One HTTP listener (host:port from preset config; bound by Node's socket-level timeouts).
- `/admin/*` routes accept JSON bodies bounded ≤ 64 KiB; `/auth/*` routes accept bodies ≤ 1 MiB (the `@ax/http-server` default).
- POST `/admin/mcp-servers/:id/test` opens an outbound MCP connection bounded by a 30-second timeout. Errors are sanitized — only `error.code` / `error.name` surfaces; no stack traces or remote response bodies bubble to the caller.
- `@ax/ipc-server` gains a `session.get-config` action authenticated by the runner's existing bearer token. The handler reads `ctx.sessionId` (set by the IPC server's auth gate after the bearer token resolves), not a body parameter; the schema is `.strict({})`. The runner cannot request another session's config.
- Rate-limit subscriber on `/auth/*` (30/min/IP) is reject-only — it cannot mutate the route table or peer-plugin state.

No FS / process-spawn / new-network reach beyond what existed pre-9.5. The `@ax/http-server` CSRF subscriber's only capability is to reject with 4xx; it cannot mutate `ctx`, the route table, or peer plugins.

### Injection

The new high-risk surfaces and how we treat them.

- **`system_prompt` is user-authored** and flows untrusted to the LLM in `chat:run` (Invariant I8). For team agents, a different user's prompt can run on this user's behalf. We brand the field as untrusted at every type definition and downstream subscriber comment, but the LLM is its sole legitimate destination. UI display before chat starts is the Week 10-12 follow-up.
- **Cookies are HMAC-SHA256-signed** with a 32-byte key from `AX_HTTP_COOKIE_KEY` (or `cookieKey` opt for tests). Tampered values return null without throwing. We use `crypto.timingSafeEqual` for the HMAC compare; HMAC-segment length-mismatch and base64url-charset mismatch short-circuit before `timingSafeEqual` so we never throw on caller-supplied input. The cookie value cap is 4 KiB.
- **CSRF guard** on POST/PUT/PATCH/DELETE: exact-match Origin allow-list OR `X-Requested-With: ax-admin` header. State-changing requests with neither return 403 `csrf-failed:{origin-missing|origin-mismatch}`. The allow-list is exact-string match — no URL parsing, no wildcards.
- **IdP errors are NEVER logged verbatim** — only `error.code` surfaces (`auth_callback_failed` event). State mismatches return 400 `callback-failed` without echoing the cookie. `dev-bootstrap` token comparison uses `timingSafeEqual` gated on a length pre-check.
- **Bodies are zod-validated** on every admin endpoint. Fields are length-capped (`displayName` 1-128, `systemPrompt` ≤ 32 KiB, `team_id` / `user_id` length-capped). `model` is allow-listed via `AX_AGENT_MODELS_ALLOWED` (env, comma-separated, with a built-in default).
- **Wildcard tool scope** (`allowedTools=[]` AND `mcpConfigIds=[]`) is the dev-only sentinel for the dev-agents stub. Production `/admin/agents` REJECTS this combination at the API (`packages/agents/src/admin-routes.ts:273`).
- **Role checks happen via SQL filter** on `teams_v1_memberships` — never trusting an actor payload alone. Role is strict-equality `'admin' | 'member'`.
- **`session_id` is 32 random bytes** via `crypto.randomBytes`; comparisons go through `@ax/http-server`'s HMAC verifier (`timingSafeEqual`).

## Cookie + CSRF posture

- **HttpOnly:** yes, always. Set unconditionally in `buildSetCookieHeader` (`packages/http-server/src/cookies.ts:154`).
- **SameSite:** `Lax` by default; if a caller explicitly passes `None`, `Secure` is forced (per spec).
- **Secure:** derived from `req.protocol` or `X-Forwarded-Proto` when `AX_TRUST_PROXY=1`; falls back to `false` for HTTP-localhost dev.
- **Signing:** HMAC-SHA256 with a 32-byte key from `AX_HTTP_COOKIE_KEY` (64 hex or 44 base64 chars). Wire format `<base64url(value)>.<base64url(hmac)>`.
- **CSRF:** on POST/PUT/PATCH/DELETE — exact-match Origin allow-list OR `X-Requested-With: ax-admin`. State-changing requests with neither return 403 `csrf-failed:{origin-missing|origin-mismatch}`. Empty Origin allow-list emits a stderr warn unless `AX_HTTP_ALLOW_NO_ORIGINS=1`.
- **Cookie value cap:** 4 KiB (storage-exhaustion guard).
- **Tamper handling:** length-mismatch and base64url-charset mismatch short-circuit before `timingSafeEqual`; tampered values return null without throwing.

## IdP allow-list

- Google OIDC configured via `AX_AUTH_GOOGLE_*` env vars in the preset (issuer, client_id, client_secret, redirect_uri).
- **Issuer URL is pinned at config time.** No auto-discovery from caller-supplied issuer. `Issuer.discover()` runs ONCE at init against `args.config.issuer` (`packages/auth/src/oidc.ts:82`).
- **Redirect URI is exact-matched.** The IdP and our config must agree byte-for-byte; `openid-client` validates this against the discovered metadata and the callback parameters.
- **Dev-bootstrap path:** refused at init when `NODE_ENV === 'production'` AND requires a non-empty `AX_DEV_BOOTSTRAP_TOKEN`. Token comparison uses `timingSafeEqual` gated on length pre-check.

## Rate-limit posture

- `/auth/sign-in/*`, `/auth/callback/*`, `/auth/dev-bootstrap`: 30/min/IP via in-memory token bucket.
- IP source: first value of `x-forwarded-for` (lowercased + trimmed) when `AX_TRUST_PROXY=1`; otherwise a fixed `'local'` bucket (degraded; documented).
- **Multi-replica limitation:** the bucket lives in process memory, so each replica has its own. Plan defers the multi-replica fix to Week 13+ via `@ax/eventbus-postgres`. For MVP this is acceptable; the abuse mitigation it provides (slowing automated dev-bootstrap brute-force) still works against single-target replicas.
