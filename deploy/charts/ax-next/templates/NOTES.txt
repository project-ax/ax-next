A few things worth double-checking on the way out:

  1. The runner namespace ({{ include "ax-next.runnerNamespace" . }}) must exist
     and the host pod's RBAC binding scopes there. The chart does NOT create
     it — `kubectl create ns {{ include "ax-next.runnerNamespace" . }}` if you
     haven't already.

  2. If gVisor is missing on this cluster, runner pod creates will fail with
     `runtimeClassName not found`. See deploy/charts/ax-next/SECURITY.md.

  3. If NetworkPolicies are disabled, runner pods can reach anything on the
     cluster. That's the kind-dev default — not a production stance.
