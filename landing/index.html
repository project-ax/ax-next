Agent infrastructure / V2.0

A nervous crab built your agent stack.

ax-next is a modular agent framework with paranoid security defaults. Bring Claude. Bring GPT. Bring a local Llama running on your gaming PC. We handle the sandboxing, capability isolation, and the part where you don't have to trust anything.

01 / Modular

Plugins all the way down.

The kernel is small. Everything else — workspaces, runners, credentials, sandboxes, transports — is a plugin. Plugins talk through hooks, not imports. Swap any of it. Build your own.

02 / Paranoid

Trust nothing. Audit everything.

Capabilities are explicit. Sandboxes are enforced. Untrusted content stays untrusted at every hop. We treat every input — including our own model outputs — as a potential bad actor.

03 / Open

Any model. Any provider.

Claude. GPT. Gemini. Mistral. A local Llama. A custom fine-tune you trained on Tuesday. ax-next doesn't care which model you run — only that you can swap it without rewiring your stack.

I.How it's built

A small kernel.
A bus of hooks.
Plugins for everything else.

The architecture is deliberately boring at the center and interesting at the edges. The kernel does almost nothing. It boots plugins, dispatches hooks, and gets out of the way.

The kernel boots plugins. Plugins talk through hooks.

A plugin is a directory with a manifest.ts, a list of hooks it provides, a list of hooks it subscribes to, and a list of capabilities it requests. That's it. No magic loaders. No DI container. No service registry.

"The hook bus IS the inter-plugin API. No cross-plugin imports. The kernel just dispatches."

Hooks are transport-agnostic and storage-agnostic by design. A workspace:read hook doesn't know whether it's hitting git, sqlite, k8s volumes, or a remote shard. Subscribers get a clean payload. Implementations get to pick their poison.

Want to swap the workspace for an in-memory shim during tests? Drop in a plugin. Want to run agents on bare metal in one environment and Kubernetes in another? Same code, different sandbox plugin. Want to bring your own credential vault? Implement credentials:get and unplug ours.

III.The five invariants

We trust nothing.
Including ourselves.

Every PR is audited against these five rules before it merges. They're uninspiring. They're rigid. They're the reason things don't catch fire when a plugin does something stupid.

i.

Hooks are transport-agnostic, storage-agnostic.

No git, sqlite, k8s, or pod_name vocabulary in hook payloads. If a field name only makes sense for one backend, it leaks the abstraction. We rename now, cheap, before subscribers depend on it, expensive.

ii.

No cross-plugin imports. Ever.

Plugins talk through the hook bus or they don't talk. Enforced by lint. The hook bus IS the inter-plugin API. If you find yourself wanting to import from a sibling plugin, the hook surface is wrong, not the import.

iii.

No half-wired plugins.

A plugin is either fully registered, fully tested, and reachable from the canary acceptance test — or it doesn't merge. "We'll wire this up later" is how you get a codebase full of code that looks done but isn't.

iv.

One source of truth per concept.

If two plugins both store state about the same thing — skills, tools, sessions — one of them is wrong. Coordinate through service hooks, not shared rows. State sprawl is how teams end up reconciling reality at 3am.

v.

Capabilities are explicit and minimized.

Every plugin, tool, IPC handler, and sandbox boundary grants the smallest set of capabilities it needs. Untrusted content — model output, tool output, plugin code — stays untrusted at every hop. The whole point of v2 is that we're the secure one.

Client power. Server safety.
Both, in one stack.

Most agent frameworks force a choice: the freedom of a hackable open client, or the security of a managed platform. We refused. ax-next gives you both — because picking one shouldn't mean losing the other.

// Like an open client

All the freedom of an unbundled hacker tool.

  • Bring your own model. Frontier API, local Ollama, fine-tune on a Mac mini — all the same to ax.
  • Plugins, not patches. Fork is a four-letter word. Extend through hooks instead.
  • Self-host the whole thing. Single binary, your hardware, your network, your rules.
  • Local inference is a first-class citizen. Air-gapped? Compliance-locked? Just paranoid? We get it.
  • Customize every surface. CLI, web, TUI, your-app-here — they're all just plugins on the same bus.
  • MCP-native. Tools speak Model Context Protocol out of the box. Bring your own.
// Like a managed platform

All the safety of a hardened enterprise platform.

  • Sandboxed execution. Every agent run lives inside a namespace cage. Bwrap on Linux, k8s pods at scale.
  • Capability isolation. Filesystem, network, spawn, env — each granted explicitly per run, never inherited.
  • Multi-tenant from day one. Workspace plugins keep state isolated. No leakage between tenants, no shared globals.
  • Credential proxy, never paste. Tokens never touch agent processes. Bridge mediates, audit logs everything.
  • Audit trail by default. Every hook dispatch is logged. Replay a session. Diff a decision. Sleep at night.
  • SSO & secret backends. OIDC, Vault, AWS Secrets Manager — all credential plugins, all swappable.
IV.What it looks like

Less than you'd think. More than you need.

A plugin is a file. A hook is a function. A capability is a list. There's no DSL, no decorators, no buildchain. Read the code. Trust what you see.

~/.config/ax/agents/research.toml TOML 
# a research agent that pairs frontier reasoning
# with a local model for cheap edits
name = "research"
workspace = "./projects/quantum-foam"

[[model]]
role = "plan"
provider = "anthropic"
model = "claude-opus-4-7"

[[model]]
role = "edit"
provider = "ollama"
model = "llama3.2:3b"
endpoint = "http://10.0.0.42:11434"

[capabilities]
fs     = ["./projects/quantum-foam"]
net    = "none"     # offline edits
spawn  = "none"
packages/runner-mything/manifest.ts TS 
import { definePlugin } from "@ax/core";

export default definePlugin({
  name: "runner/mything",
  version: "0.1.0",

  // what we promise to provide
  provides: [
    "runner:start",
    "runner:cancel",
  ],

  // what we listen to
  subscribes: ["workspace:changed"],

  // minimum needed; kernel rejects extras
  capabilities: {
    net: ["api.mything.dev"],
    fs:  "workspace-only",
  },

  async setup(ctx) {
    ctx.on("runner:start", handleStart);
  },
});
