# Project context

Confirmed facts about ax-next. Not opinions.

- `2026-04-23` — Only `@ax/cli` and `presets/**` (the meta-packages that wire plugins together) may import sibling plugins; enforced by `eslint.config.mjs` `no-restricted-imports` overrides. All other inter-plugin communication goes through the hook bus. (Updated 2026-05-24: `presets/**` is on the allow-list too, not just the CLI.)
- `2026-04-23` — Storage hook payload contract is `{ key: string, value: Uint8Array }` — opaque bytes, no backend vocab. Any `table`/`bucket`/`rowid` field would leak.
- `2026-04-23` — pnpm 10 requires `pnpm.onlyBuiltDependencies` allowlist for packages with postinstall scripts (e.g. `better-sqlite3` prebuild-install). Without the allowlist, the build silently skips the postinstall and the package fails to load at runtime.
- `2026-04-23` — E2E tests invoke the CLI as `node dist/main.js <args>`, not `pnpm exec ax-next`. Shebang + `chmod +x` are present but tests use node directly to avoid pnpm bin-link flakiness.
- `2026-04-23` — Week 3 shipped as PR #2 (merged). Week 1–2 kernel shipped as PR #1.
- `2026-04-29` — Claude Agent SDK 0.2.119 encodes the project-dir name as `realpath(cwd)` with `/` → `-`, NOT the literal cwd string. On macOS `/var/...` becomes `-private-var-...`. Don't reconstruct the path; glob `<HOME>/.claude/projects/*/<sessionId>.jsonl`.
- `2026-04-29` — SDK 0.2.119 with `settingSources: []` still writes outside `<HOME>/.claude/projects/`: `<HOME>/.claude.json` (~20KB cache: GrowthBook flags, anon userID, migrationVersion, model-cost cache), `<HOME>/.claude/backups/.claude.json.backup.<ts>` (accumulates per spawn), `<HOME>/.claude/policy-limits.json`. None are secrets, but workspace persistence should ignore everything outside `.claude/projects/`.
- `2026-04-29` — SDK 0.2.119 jsonl entry types include non-turn records: `queue-operation`, `last-prompt`, `attachment` (e.g. `skill_listing` injected by the SDK even with `settingSources: []`). Any transcript parser must whitelist `user`/`assistant`/`tool`, not blacklist. (Updated 2026-05-24: the once-planned `runner:read-transcript` hook never shipped — Phase C was superseded by the 2026-05-01 workspace redesign. Transcripts are now read by `conversations:get` globbing committed `.claude/projects/*/<sessionId>.jsonl`. Stale code-comments still mention the hook in `packages/conversations/src/{types,store}.ts`.)
- `2026-04-29` — ax-next has no production data and never will need a schema migration that preserves rows. Existing `_v1` table suffixes (`conversations_v1_*`, `audit_v1_*`) are stable identifiers, NOT version pointers. ALTER TABLE in place is correct for every schema change; no v1→v2 split, ever. (See decisions.md 2026-04-29 entry.)
- `2026-05-03` — Host-pod listener layout (issue #39, PR #40): `@ax/ipc-http` on containerPort 8080 (runner-pod back-channel; Service :80) AND `@ax/http-server` on containerPort 9090 (public — `/chat`, `/health`, `/admin/*`, `/auth/*`, `/api/chat/*`; Service :9090). `serve` registers `/chat` + `/health` against http-server via `http:register-route`; no third listener. Chart values: `host.ports.ipc` / `host.ports.publicHttp` / `host.ipcServicePort` / `host.publicHttpServicePort`.
- `2026-05-03` — POST `/chat` (and any other state-changing route on the public surface) is gated by `@ax/http-server`'s CSRF subscriber. Headless callers must send `X-Requested-With: ax-admin` OR an Origin in `AX_HTTP_ALLOWED_ORIGINS`. The chart sets `AX_HTTP_ALLOW_NO_ORIGINS=1` to silence the empty-allow-list warning when no Origin allow-list is configured.
