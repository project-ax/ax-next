The deterministic fix instead keys on the **`.ax/uploads/` attachment namespace** (our own convention, which the model can't strip): a PreToolUse `resolveAttachmentPaths` re-roots the structured path fields (`file_path`/`path`/`notebook_path`) whose value references the `.ax/uploads/` segment to `<workspaceRoot>/.ax/uploads/‚Ä¶`, BEFORE the host `tool:pre-call` so the host adjudicates the real path (and refusing any `..` segment). Two reusable facts: (a) **the SDK PreToolUse hook can REWRITE tool input** via `hookSpecificOutput.updatedInput` (works for builtins like Read); (b) **only a live cluster reveals how the model handles a path/scheme** ‚Äî Playwright masks file dialogs AND you can't unit-test model path-resolution behavior, so a model-facing path change MUST be walked on kind, not just unit-tested.
