# Patterns

## Patterns (do these)

- `2026-04-23` ā€” When adding a new service hook, fill in the boundary-review block in the PR description (alternate impl / leaky field names / subscriber risk / wire surface). If you can't name an alternate impl, it probably shouldn't be a hook yet.
- `2026-04-23` ā€” When scaffolding a plugin that touches fs/net/process/untrusted input, drop a `SECURITY.md` in the package directory at scaffold time, not later. Week 3's `packages/storage-sqlite/SECURITY.md` is the template.
- `2026-04-23` ā€” For ESM "am I the main module?" checks, use `import { pathToFileURL } from 'node:url'` and compare `import.meta.url === pathToFileURL(process.argv[1]).href`. `require.main === module` doesn't exist under ESM.
- `2026-04-29` ā€” When reading a Claude Agent SDK jsonl by sessionId, glob `<HOME>/.claude/projects/*/<sessionId>.jsonl` rather than reconstructing the encoded-cwd directory. The SDK encodes `realpath(cwd)`, not the literal cwd, so any symlink (e.g. macOS `/var` ā†’ `/private/var`) breaks reconstruction. SessionId is unique across project dirs.
- `2026-04-29` ā€” When parsing SDK jsonl, whitelist turn-bearing entry types (`user`/`assistant`/`tool`) rather than blacklisting bookkeeping types. The SDK adds new bookkeeping kinds (`queue-operation`, `last-prompt`, `attachment`) over time; a blacklist drifts.
- `2026-05-03` ā€” When a PR wires a new plugin into the k8s preset, the chart-shape test (`deploy/charts/ax-next/__tests__/env-shape.test.ts`) is the contract that catches missed env-var stamping. The test scans `presets/k8s/src/index.ts` for `env.*` reads and asserts both directions against the rendered deployment. Run it; if it fails, the chart's env block is incomplete ā€” fix in the same PR.
- `2026-05-03` ā€” Subcommands that need an HTTP front door should register routes via `http:register-route` against `@ax/http-server`, not spin up their own `http.createServer`. Mirror the channel-web / auth-oidc / teams pattern: a plugin with `manifest.calls: ['http:register-route']`, routes registered in `init()`. Multiple listeners on the same pod cause `EADDRINUSE` collisions and bypass the http-server's CSRF / body-cap subscribers.
- `2026-05-24` ā€” Adding a NEW field to `PluginManifestSchema` must be `.optional()` (NOT `.default([])`) if you don't want to touch every existing in-code manifest. `PluginManifest = z.infer<schema>` uses the schema's *output* type, and a `.default([])` field is **required** in the output type ā€” so all ~20 hand-authored manifest literals fail tsc with "Property X is missing". `.optional()` makes it `X?:` in the inferred type (additive, zero churn); have `bootstrap` treat the absent value as empty (`manifest.optionalCalls?.map(...) ?? []`). This is why `calls`/`registers`/`subscribes` have `.default([])` yet don't error ā€” every manifest already spells them out explicitly.
- `2026-05-24` ā€” `pnpm` is NOT on the orchestrated-agent PATH; only `node` + `corepack` are (under `~/.nvm/versions/node/<v>/bin`). Run pnpm as `corepack pnpm <args>`. But the root `test` script does `pnpm -r run test` internally, and that nested invocation can't find `pnpm` ā†’ `sh: 1: pnpm: not found`. Fix: drop a 2-line wrapper (`#!/usr/bin/env bash\nexec corepack pnpm "$@"`) into a tmp dir and prepend it to PATH so nested `pnpm` resolves.

## Anti-Patterns (don't do these)

- `2026-04-23` ā€” Don't put backend-specific field names in hook payloads (`sha`, `bucket`, `pod_name`, `table`, `rowid`). If the name only makes sense for one impl, rename before subscribers depend on it.
- `2026-04-23` ā€” Don't land "we'll wire it later" infrastructure. Either a live caller reaches it in the same PR or it doesn't merge. Symmetric API pairs (`get`/`set`) with a documented near-term consumer are the narrow exception.
